# 贡献指南

感谢你愿意参与“迟暮的泪”！无论你是开发者、设计师、硬件极客还是普通用户，都能找到贡献的方式。

## 行为准则

请遵守 [Contributor Covenant](https://www.contributor-covenant.org/) 2.1 版，并特别尊重“伦理框架”中的原则。

## 如何贡献

### 代码与文档

1. 从 [Issues](https://github.com/your-repo/tearsofnostalgia/issues) 选择一个未分配的任务，或提出新 Issue。
2. Fork 仓库，创建分支 `feature/描述` 或 `fix/描述`。
3. 提交代码前运行测试（如果有）并更新相关文档。
4. 发起 Pull Request，等待至少一位维护者审核。

### 硬件与实地测试

- 如果你有 MR 设备或传感器，可以在 [Discussions](https://github.com/your-repo/tearsofnostalgia/discussions) 中发布兼容性报告。
- 如果你愿意提供测试场地（如养老院、博物馆），请通过邮件联系项目组。

### 伦理与法律

- 如果你有法学、伦理学背景，欢迎帮助完善 ETHICS.md 或设计审查流程。
- 如果你发现某个应用可能违反伦理框架，请通过 Issue 报告。

## 开发环境设置

快速体验 WebXR 原型：只需一个 HTTPS 静态服务器（可以使用 `npx serve` 或 GitHub Pages）。

完整开发环境（需要 MR 设备）：
- Unity 2022 LTS + Mixed Reality Toolkit (MRTK3)
- Visual Studio 2022
- Windows 10/11

## 任务标签说明

- `good first issue`：适合首次贡献者
- `help wanted`：需要额外人手
- `ethics`：涉及伦理审查
- `hardware`：需要真实设备测试
